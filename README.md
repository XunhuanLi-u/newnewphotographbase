# Portrait Practice Lab

一个本地运行的人像练习网站。你可以选择内置参考构图，也可以上传自己的模板照片和备注，让网页生成拍摄建议、后期方向和可套用的练习预设。

## 使用方式

直接用浏览器打开 `index.html`。

照片处理和本地分析都在浏览器内完成。练习作品、自定义模板和接口地址会保存到当前浏览器的 `localStorage`；API Key 只在本次页面里使用，不会写入本地存储。

## 已包含功能

- 四个原创参考模板：伦勃朗光、窗光三分法、低调侧脸、金色逆光居中
- 模板库分档管理：可新建、重命名、删除模板库；项目默认模板库也支持本地调整后导出为上线模板包
- 模板库上线：可导出浏览器里的自定义模板数据，并整理成 `assets/template-libraries` 静态资源
- 自己上传模板照片，填写模板名称和备注
- 上传模板照片后优先读取 EXIF 原片参数：ISO、快门、光圈、焦距、拍摄时间、相机、镜头、曝光补偿、白平衡
- 如果 EXIF 缺失，回退到本地视觉分析建议：构图重心、影调、反差、冷暖、拍摄参数、后期方向
- 可选接入 OpenAI 兼容的视觉模型接口，生成更细的构图和影视后期分析
- 上传练习照后自动分析曝光、白平衡和反差
- 根据所选模板自动生成后期效果
- 下载成片并保存到私人练习相册

## API 说明

模板分析区支持 OpenAI 兼容的 Chat Completions 视觉输入格式：把模板图压缩为 base64 data URL 后发送给模型，同时把 EXIF 原片参数、用户备注和本地视觉分析结果放进提示词。静态网页直接请求 API 时，可能遇到浏览器 CORS 或密钥暴露问题；正式部署时更适合通过自己的后端或代理转发。

## 把自定义模板一起上线

网页里上传的模板默认保存在当前浏览器的 `localStorage`，不会自动变成 GitHub 仓库文件。要把它们作为默认模板库一起上线，有两种方式：

### 方式一：直接从网页导出上线包

1. 在网页里整理好模板库，删除、重命名或新增模板。
2. 点击模板库工具条里的 `导出上线包`。
3. 把下载到的 `manifest.js` 和 `manifest.json` 覆盖到 `assets/template-libraries/`。
4. 把这些文件上传到 GitHub。

网站会优先读取 `assets/template-libraries/manifest.js`，所以上传后这些模板就会变成随网站一起上线的默认模板。

### 方式二：导出数据后用脚本整理图片文件

1. 打开网站，点击模板库工具条里的 `导出数据`，下载 `portrait-template-export.json`。
2. 把这个 JSON 放到项目根目录。
3. 运行：

```bash
node tools/materialize-template-export.mjs portrait-template-export.json
```

脚本会把模板图片按模板库分类写入 `assets/template-libraries/`，并生成 `manifest.js` 和 `manifest.json`。把这些文件一起提交到 GitHub 后，线上网站会自动读取这些模板库。
